## 📋 项目简介

本项目基于 STM32F103 平台，从裸机开发演进到 FreeRTOS 多任务系统，实现了一套包含**计算器、贪吃蛇游戏、LED 控制、串口命令解释器**的多功能嵌入式应用。项目采用 **Device → Middle → APP → UI** 四层分层架构，以**状态机 + 事件驱动**为核心设计思想，通过 FreeRTOS 队列实现任务间通信。

> 🔗 GitHub 仓库：[https://github.com/DingDa-H/stm32f1_rtos](https://github.com/DingDa-H/stm32f1_rtos)

## 🧱 软件架构

```
┌─────────────────────────────────┐
│           UI 层 (Ui/)            │  ← 菜单系统、UI 按钮控件、文本控件
├─────────────────────────────────┤
│          APP 层 (APP/)           │  ← 贪吃蛇、LED 控制、命令解释器、计算器
├─────────────────────────────────┤
│        Middleware 层 (Middle/)    │  ← 按键事件中间件
├─────────────────────────────────┤
│         Device 层 (Device/)      │  ← 按键/OLED/矩阵键盘/LED/环形缓冲驱动
├─────────────────────────────────┤
│   HAL 层 (Core/ + Drivers/)     │  ← STM32CubeMX 生成 + HAL 库
└─────────────────────────────────┘
```

### 架构要点

- **Device 层**：封装硬件寄存器操作，向上提供统一接口（如 `emBtnGetCurEvent()`）
- **Middleware 层**：处理底层数据到应用层语义的转换，如按键消抖状态机
- **APP 层**：实现业务逻辑，不直接操作硬件寄存器
- **UI 层**：管理页面切换、控件渲染，与 APP 层通过回调函数解耦

## 🧵 FreeRTOS 任务设计

| 任务名 | 优先级 | 栈大小 | 周期 | 职责 |
|--------|--------|--------|------|------|
| `BtnScan` | 3 | 128 words | 10ms | 物理按键 + 4×4 矩阵键盘扫描，消抖处理后通过队列上报事件 |
| `AppTask` | 4 | 256 words | 事件驱动 | 阻塞等待按键队列，根据当前页面分发事件到对应处理器 |
| `Serial` | 2 | 128 words | 事件驱动 | 阻塞等待任务通知，唤醒后轮询串口帧解析 |
| `Display` | 1 | 200 words | 30ms | 周期性刷新 OLED 显存缓冲区 |

### 任务间通信

```
BtnScan ──(Queue)──▶ AppTask ──(函数调用)──▶ Menu / Snake / Calculator
Serial  ◀──(TaskNotify)── Menu（进入串口页面时通知唤醒）
```

- **按键队列**：容量 10，非阻塞发送（队列满时丢弃，保证扫描周期不被破坏）
- **任务通知**：串口任务在不活跃时挂起（0 CPU 开销），进入串口页面时由菜单系统唤醒

## 🔄 状态机设计

### 1. 物理按键状态机（底层驱动）

```
   ┌──────────┐  按下+消抖  ┌──────────┐
   │ Released │ ──────────▶ │ Pressed  │
   │          │ ◄────────── │          │
   └──────────┘  释放+消抖  └──────────┘
        │                        │
        ▼                        ▼
   Click / DoubleClick      LongPress
```

- 消抖时间可配置（默认 20ms）
- 支持单击 / 双击 / 长按三种事件类型
- 双击等待窗口可配置，长按判定在按键释放时触发

![按键状态机](https://github.com/user-attachments/assets/afd112e3-0167-41f1-8630-13f7988c0d9f)

### 2. 薄膜 4×4 矩阵按键状态机

```
  ┌──────────┐  检测到按键  ┌──────────┐
  │ 等待按下  │ ──────────▶ │ 等待松开  │
  │          │ ◄────────── │ (消抖3次) │
  └──────────┘  全键释放    └──────────┘
```

- 行列扫描后全键读取，避免"鬼键"问题
- 松手消抖计数 3 次（约 50ms），防止连击

## 📦 功能模块详解

### 🧮 计算器
- 基于状态机实现加减乘除四则运算
- 4×4 薄膜矩阵按键输入（0-9、+、-、×、÷、=、删除、清零）
- OLED 实时显示输入和结果
- 支持连续运算

### 🐍 贪吃蛇
- **静态内存池 + 双向链表** 管理蛇身（避免动态分配碎片化）
- 双向链表使尾部删除为 O(1)，优于单向链表的 O(n)
- 碰撞检测：墙壁 + 自身
- 食物随机生成（避免与蛇身重叠）
- 游戏状态机管理：空闲 → 运行 → 胜利/失败

### 💡 LED 控制
- 支持 4 路 LED 独立开关/翻转
- 通过 UI 按钮控件操作（子菜单页面）
- 同时注册到命令解释器，支持串口指令控制

### 🖥 串口命令解释器
- **动态命令注册**机制：各应用模块通过 `vInterpreterRegisterCmdTable()` 注册命令
- 命令表格式：`对象 + 命令 + 参数个数 + 回调函数`
- 串口帧协议：以 `\n` 结尾，支持 `LED On 0` 等指令
- 独立帧缓冲区，环形缓冲接收

### 📟 OLED 显示驱动
- 软件模拟 I2C 通信
- 显存缓冲区机制，支持多点绘制后统一刷新
- 支持 8×16 / 6×12 两种字体
- 支持正显 / 反显模式

### 🎛 二级菜单系统
- 主菜单：计算器、LED 控制、贪吃蛇、串口测试
- 子菜单页面独立管理按键事件
- 窗口滚动机制：菜单项超过屏幕可见行数时自动滚动

## 🚀 编译与运行

1. 使用 STM32CubeMX 打开 `001oled_keypad_calculator.ioc` 生成初始化代码
2. 使用 Keil MDK-ARM 打开 `MDK-ARM/` 目录下的工程文件
3. 编译下载到 STM32F103 开发板
4. 串口工具连接 UART1（115200bps），发送命令如 `LED On 0`

## 🎯 技术亮点

1. **分层解耦**：Device → Middle → APP → UI 四层架构，每层职责清晰，便于移植和维护
2. **状态机驱动**：按键、计算器、贪吃蛇均采用状态机建模，逻辑清晰，边界条件完整
3. **事件驱动 IPC**：基于 FreeRTOS 队列的发布-订阅模式，按键扫描任务与业务任务解耦
4. **零拷贝任务通知**：串口任务使用 `TaskNotify` 替代信号量，空闲时完全挂起
5. **静态内存管理**：全部使用静态数组/内存池，无 `malloc`，适合资源受限的嵌入式场景
6. **双向链表优化**：贪吃蛇从环形缓冲区 → 单向链表 → 双向链表逐步重构，尾部删除从 O(n) 优化到 O(1)
